# Navik — навигатор на Java (Phase 1)

Открытый навигатор в духе Google Maps/Waze, написанный полностью на **Java**
(Kotlin DSL используется только для конфигурации Gradle-сборки, как и требовалось).

## Что уже работает (Phase 1)

- Карта на весь экран на **MapLibre GL** (векторные тайлы OpenStreetMap), зум/поворот/наклон/компас.
- Поиск адресов и организаций через **Nominatim** + **Photon** (оба сервиса бесплатны).
- Построение маршрута **А → Б** через **OSRM** (профили: авто / пешком / велосипед),
  альтернативные маршруты, отрисовка на карте, ETA и расстояние.
- Экран пошаговой навигации с **голосовыми подсказками** (Android TextToSpeech).
- Локализация: **русский / украинский / английский**.
- Светлая/тёмная тема (Material 3).
- Архитектура: **MVVM**, **Room** (готова для избранного и истории), **LiveData**, **ViewModel**,
  Repository-слой, разделение на пакеты `data` / `ui` / `util`.

## Что ещё предстоит (следующие фазы)

Это не демо, а честный работающий каркас — но полный паритет с Google Maps/Waze
физически не помещается в одну итерацию. Не реализовано пока:

- Краудсорсинг событий Waze-стиля (ДТП, камеры, полиция, оценка сообщений).
- Настоящие офлайн-карты (сейчас — только сетевые тайлы, без офлайн-кэша MBTiles/PMTiles).
- 3D-здания, подсказки по полосам движения, отображение реальных пробок.
- Общественный транспорт (OSRM его не считает — нужен отдельный движок вроде OpenTripPlanner/Motis).
- Автоматическое отслеживание прогресса по GPS во время навигации (сейчас шаг переключается вручную —
  тап по экрану). Автоматика требует map-matching (привязку GPS-точки к дороге).
- UI избранного/истории/дома/работы (DAO и Room-таблицы уже готовы, экранов пока нет).
- Виджет, запись GPS-трека, импорт/экспорт маршрутов, шеринг.

## Как поставить свою иконку приложения

Сейчас стоит временная заглушка — буква "N" на бирюзовом фоне
(`app/src/main/res/drawable/ic_launcher_foreground.xml` и `ic_launcher_background.xml`).

Проще всего заменить через Android Studio:

1. Правой кнопкой на `app` → **New → Image Asset**.
2. Icon Type: **Launcher Icons (Adaptive and Legacy)**.
3. Загрузите свою картинку (лучше PNG/SVG с прозрачным фоном, квадрат ≥512×512).
4. **Finish** — Android Studio сам сгенерирует и заменит все нужные файлы
   (`mipmap-anydpi-v26`, `mipmap-hdpi`, `mipmap-xhdpi` и т.д.), заглушку можно будет удалить.

Вручную это тоже можно сделать — просто положите свои PNG в папки
`app/src/main/res/mipmap-mdpi/`, `mipmap-hdpi/`, `mipmap-xhdpi/`, `mipmap-xxhdpi/`,
`mipmap-xxxhdpi/` под именами `ic_launcher.png` и `ic_launcher_round.png`,
и при желании удалите `mipmap-anydpi-v26/ic_launcher.xml` (adaptive-версию).

## Как собрать APK через GitHub Actions

1. Создайте новый репозиторий на GitHub.
2. Загрузите содержимое этого архива в репозиторий (включая `.github/workflows/build-apk.yml`).
3. Запушьте в ветку `main` — сборка запустится автоматически
   (или запустите вручную: вкладка **Actions → Build Debug APK → Run workflow**).
4. После завершения сборки зайдите в workflow run → **Artifacts** → скачайте `naviapp-debug-apk`.
5. Внутри архива будет `app-debug.apk` — устанавливайте на телефон.

## Как открыть в Android Studio локально

В репозитории **нет** бинарного `gradle-wrapper.jar` (бинарники плохо переживают
передачу через чат/архив). При первом открытии проекта в Android Studio:

- Android Studio предложит **"Create Gradle Wrapper"** — согласитесь, либо
- выполните `gradle wrapper --gradle-version 8.9` в терминале проекта (нужен
  локально установленный Gradle либо Android Studio сама его подтянет).

После этого проект синхронизируется и собирается стандартным способом
(**Build → Build Bundle(s) / APK(s) → Build APK(s)**).

## Использованные бесплатные сервисы

| Функция | Сервис | Требует API-ключ |
|---|---|---|
| Тайлы карты | MapLibre demo style (OSM) | Нет |
| Геокодирование адресов | Nominatim | Нет (есть лимит 1 запрос/сек) |
| Поиск организаций/POI | Photon (Komoot) | Нет |
| Построение маршрутов | OSRM demo-сервер | Нет |

⚠️ Все три сервиса — **публичные демо-инстансы**, не предназначенные для
промышленной нагрузки. Для реального продакшена нужен self-hosted OSRM/Nominatim
или платный тариф (MapTiler, Geoapify, Mapbox и т.п.) — см. `RetrofitClient.java`,
там меняется один `BASE_URL`.
